What is a Workflow?

Workflows = Claude running a team of AI agents

When a task is too big for one conversation, Claude can launch a workflow — a script that spins up many AI sub-agents working in parallel, each with a specific job. They share findings, critique each other, and synthesize results.

Think of it like Claude hiring a research team, a panel of critics, and a writing committee — all running at the same time.

They run "in the background"

The workflow keeps running even while you type. Claude says "I'll have the report when it completes" and you can check progress with /workflows in the Claude Code terminal.

Important   "Background" means within the same Claude Code session. If that session ends or restarts — the workflow is killed.

What To Do Next

Re-launch the workflow (it will start fresh)

  1. Just say "re-launch it" — Claude will re-run the same workflow script from scratch.
  2. The workflow will take 15–30 minutes to complete (it's doing a lot of parallel research and multi-round critique).
  3. Don't close the session while it runs — or it will be killed again. Keep the Claude Code window open.
  4. When it finishes, Claude will get a notification and write the final spec to a file you can read.

Alternatively: faster, simpler version

If you don't want to wait 30 minutes, Claude can do a lightweight version — research only (6 agents, ~5 min) and write a spec directly without the 3-round critique loop. Faster but less thoroughly stress-tested.
