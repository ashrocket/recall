## What happened

Ashley asked for "a restart prompt using recall and the local llm" at the end of a design-system session (palette audit, muted-text consolidation, PR #[N] merged). I saved a checkpoint, discovered the local/LLM comparison workflow by reading `recall-save-eval.py` source, wrote the LLM candidate, logged the comparison with `--winner llm`, and the registry promotion + `restart --launch` both worked first try (modulo one flag-order stumble, below).

## What worked well

1. **The candidate → log → promote handshake is a genuinely good design.** `candidate` hands you a collision-safe path, `log` records an honest JSONL comparison (`recall-save-evals.jsonl`), and winner=llm atomically swaps the registry pointer. `restart --launch` then used the promoted prompt with zero extra steps.
2. **Honest provenance.** The local prompt says "No LLM distillation was used" right at the top. That one line is what triggered the whole improvement loop — keep it.
3. **The wrapper's RECALL_ROOT discovery** (claude + codex cache paths, version-sorted, exec) worked unmodified, and checking the codex path signals cross-agent intent that matches how Ashley actually wants to use restarts.
4. **Editing the prompt file after promotion still works** — the registry stores the path, so post-hoc improvements (I added grep recipes + a Codex section) flow into `restart --launch` for free. Nice property; worth documenting as intended.
5. **`save` indexing is fast** and the console summary (messages/commands/failures/skills counts) is the right amount of info.

## Friction and bugs, in severity order

1. **The LLM comparison workflow is invisible.** `recall help` never mentions `recall-save-eval.py`, `candidate`, `log`, or promotion. I only found it by grepping the plugin source for "llm". The flagship quality feature is undiscoverable — an agent that doesn't go source-diving will ship the TF-IDF prompt every time. Suggestion: `recall save --llm` (or a post-save hint line: "To distill: …candidate/log…") that walks the calling agent through the handshake.
2. **`restart <name> --launch` fails; only `restart --launch <name>` works.** Trailing flags get swallowed into the search text: `No entries matching 'figma-palette-muted-text --launch'`. Natural typing order shouldn't matter — strip recognized flags from anywhere in argv.
3. **Local prompt quality: the extractor has no junk filter.** Top "Key Signal" for this session was a pasted terminal spinner (`vitating… (14m 46s · ↓ 33.6k tokens) Tip: Run /install-github-app…`). TF-IDF loves rare tokens, and terminal noise is maximally rare. Cheap wins: drop lines matching spinner/tip/token-count patterns, and de-prioritize lines that were pasted UI rather than typed intent.
4. **Local prompt captures activity, not decisions.** It listed commands and files faithfully but carried zero of the session's three ratified decisions — the single most valuable thing for a restart. Even without an LLM, lines where the user says "decided/let's go with/merge it" are extractable signal.
5. **Name vs slug mismatch.** `save figma-palette-muted-text` registered the name correctly but derived the prompt filename from the session title (`figma-use-find-css-design.prompt`). Harmless, but confusing when you go looking for the file — consider naming the file after the given name.
6. **`[ungrouped] theme: Mariana` in `restart list`** — no idea what Mariana is (terminal theme? group label leak?). Reads as debug output; either label it or hide it.

## Codex portability verdict (Ashley asked)

The local prompt alone would not be enough for Codex — it assumes CLAUDE.md context and captures no decisions. The LLM prompt, after two additions (regeneration grep recipes; a "non-Claude agents" section carrying the CLAUDE.md facts: two vitest configs, dev auto-deploys, PRs base dev), is agent-agnostic. Recommendation for recall: a `--portable` save mode that reminds the prompt author to inline anything that normally arrives via CLAUDE.md/MCP/memory, since Codex reads AGENTS.md and has none of those.
